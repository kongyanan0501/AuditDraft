# Demo 脚本 — 安永华明 AI 赛版

> 目标时长 **3–5 分钟**。  
> 统一口径：**费用循环 AI 协审**（初筛 + 可复核草稿底稿），**不替代**注册会计师判断。  
> 赛前检查：[`ey-demo-runbook.md`](./ey-demo-runbook.md)  
> 路演 PPT + 讲稿：[`ey-pitch-script.md`](./ey-pitch-script.md) · [`pitch/AuditDraft-EY-Pitch.pptx`](./pitch/AuditDraft-EY-Pitch.pptx)

---

## 0. 开场前 5 分钟

按 runbook 勾选。推荐演示日默认：

- 主路径：`AUDIT_DEGRADED_MODE=rules_only`（防额度翻车；规则故事完整）  
- 或完整模式（LLM 额度充足时）  
- 备用页：`/eval`、`/demo/snapshot`

主数据：`samples/ey_expense_demo_3k.csv`（约 3000 行，4 类埋雷）。  
小样对照：`samples/expense_transactions.csv`（11 行，适合快速重跑）。

## 1. 时间盒流程（含话术）

### Step 1 · 登录（10s）
- 操作：全新账号登录。
- 话术：「多租户隔离，Supabase RLS，按项目组用户隔离数据——符合事务所数据边界习惯。」

### Step 2 · 上传（15s）
- 操作：上传 `ey_expense_demo_3k.csv`（或小样）。
- 话术：「费用明细往往上千行。传统靠助理抽查；我们让确定性规则全量初筛，把人的时间留给判断与复核。」

### Step 3 · 工作流（20s）—— 亮点：编排 + 混合推理
- 操作：指 Workflow Tracker。
- 话术：「LangGraph 七步：解析 → 计划 → **规则引擎** → 异常 → 风险 → 底稿 → 导出。  
  **硬结论只来自规则，不依赖大模型**；大模型只做理解与成文。这是可治理的混合推理，不是黑盒扫表。」

> 若开了降级：坦然说「今日演示开启规则-only，保证可复现；完整模式再叠加 LLM 底稿润色。」——这是加分，不是减分。

### Step 4 · 风险卡（30s）
- 操作：展示 high/medium 卡片；可对照 `EY_DEMO_PLANTED_RISKS.md`。
- 话术：「重复付款、无审批大额、拆分报销、金额离群——都是费用循环常见红旗。同输入同输出，可复现、可上复核底稿。」

### Step 5 · 可解释四要素（40s）—— 核心差异
- 操作：展开任一 Risk Card。
- 话术：「每条结论四要素：触发规则 / 数据证据 / 准则引用 / 解释。  
  **没有证据，就没有结论。** 这才能进 Prepare → Review 链条，而不是聊天窗口截图。」

### Step 6 · 可信证据页（30s）—— 技术评委
- 操作：打开 `/eval`。
- 话术：「离线 Golden Set：召回 / 精确率有数字，且不依赖大模型。  
  旁边是人工 vs 系统耗时与命中对比——实用性用证据说话。」

### Step 6b · 调参 / 勾稽（选讲 20s）
- `/settings/rules`：改审批阈值，findings 即时变  
- `/reconcile`：报销↔付款多源勾稽  
- 被点名挑战：`/challenge` 上传评委 CSV

### Step 7 · 底稿 + 导出（15s）
- 操作：底稿折叠区 +（完整模式）导出 Word/PDF。
- 话术：「输出明确标注 AI-assisted draft，须项目组复核。我们赋能项目组，不替代签字合伙人。」

### 收尾（20s）
1. 效率：全量初筛秒级 / 分钟级，而非数小时翻表。  
2. 可靠：规则确定性 + 可离线评测。  
3. 可解释：四要素进复核。  
4. 可治理：降级模式、版本化规则、模块可替换。

## 2. 话术红线（禁止）

| 不要说 | 改成 |
| --- | --- |
| 替代审计师 / 自动出审计意见 | 辅助初筛与底稿起草，结论需复核 |
| 100% 发现舞弊 | 提升红旗检出效率，误报可配置 |
| 颠覆审计 | 规模化赋能费用循环协审 |
| 黑科技 LLM 审计 | 确定性规则 + 生成式表达 |

---

## 3. 故障预案（必练一次）

| 现象 | 动作 | 话术 |
| --- | --- | --- |
| LLM 403 / 额度耗尽 | 确认 `AUDIT_DEGRADED_MODE=rules_only` 并重启；重试任务 | 「硬检测不依赖模型；今日走规则-only 降级」 |
| 任务一直 failed | 打开 `/demo/snapshot` | 「这是官方 failover 快照，非造假 live；展示四要素与底稿骨架」 |
| 上传失败 | `/eval` 讲数字 + 快照讲风险 | 「评测与规则内核离线可证」 |
| 被问会不会编造 | 指四要素 + Golden Set + 降级仍出规则结论 | 「LLM 不能新增无证据风险」 |

## 4. 评委追问速查（安永向）

| 提问 | 应答要点 |
| --- | --- |
| 能进项目组用吗？ | 定位协审初筛；输出 draft；需 Review；下一步做复核流与重要性（路线图包 B） |
| 和 ChatGPT 贴表有何不同？ | 规则保确定性 + 四要素 + 离线评测 + RLS；不是对话生成意见 |
| 数据出境 / 安全？ | 演示用脱敏合成数据；生产按用户 RLS；service role 仅服务端 |
| 误报怎么办？ | 规则可版本化、可调阈值；负例进 Golden Set 回归 |
| 为何用 LangGraph？ | 审计是可编排质量流程，节点可测、可降级、可插人工门闩 |
